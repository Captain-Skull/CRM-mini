## 3. Бизнес цель и KPI

**Бизнес-цель:** повысить конверсию лидов в продажи и сократить время этапов.

| KPI | Формула | Ед. изм. | Целевое значение | Способ измерения |
|-----|---------|----------|------------------|------------------|
| **Конверсия по стадиям** | (Количество лидов, прошедших стадию X) / (количество лидов на предыдущей стадии) | % | ≥ 60% | Отчет "Воронка" в 1С |
| **Конверсия "new -> won"** | (Количество лидов в статусе won) / (количество лидов в статусе new) | % | ≥ 20% | Отчет "Воронка" в 1С |
| **Средняя длительность этапов** | ∑(left_at - entered_at) / (количество лидов) | дни | ≤ 3 дня | Расчёт на основе регистра истории статусов |

## 4. Границы MVP (scope/out of scope)

**Scope:**
- Справочник «Лиды» с полями: источник, дата создания, текущий статус, ответственный.
- Импорт данных из CSV в справочники и регистр истории статусов (идемпотентно).
- Экспорт данных в CSV.
- Прототип канбан-доски для визуального перемещения лидов по статусам (UI-макет).
- Отчёт «Воронка продаж» с отображением количества лидов на каждом этапе и конверсии с возможностью фильтрации.
- Расчёт KPI (конверсия и длительность) с выводом на дашборд.
- Мок-REST API (эндпоинт `/leads`) для демонстрации интеграции.

**Out of scope:**
- Интеграция с реальной телефонией, почтой, мессенджерами.
- Работа с персональными данными (ПДн) – только синтетика.
- Адаптивный интерфейс (без мобильной версии).

## 5. Данные

| Параметр | Описание |
|----------|----------|
| **Источник** | Синтетические данные, генерируемые по схеме из приложения C |
| **Форматы** | CSV (кодировка UTF-8, `,` разделитель, `"` кавычки) |
| **ОбЪем** | T1: ≥ 100 лидов, ≥ 300 записей истории статусов. T2: ≥ 1000 лидов |
| **Основные таблицы** | `leads.scv (ld, firstname, lastname, phoneNumber, email, stage, sourceId, managerId, createdAt, enteredAt)`, `lead_stages.csv (id, leadId, stage, enteredAt, leftAt)` |
| **Валидации** | порядок стадий, entered_at < left_at, домен stage |
