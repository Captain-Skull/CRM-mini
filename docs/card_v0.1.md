## 2. Бизнес-цель и KPI
**Бизнес-цель:** повысить конверсию лидов в продажи и сократить время прохождения этапов воронки.

| KPI | Формула | Ед. изм. | Целевое значение | Способ измерения |
|-----|---------|----------|-------------------|------------------|
| Конверсия «new → won» | (Кол-во лидов в статусе won) / (Кол-во лидов в статусе new) * 100 | % | ≥ 20% | Отчёт «Воронка» в 1С |
| Средняя длительность этапа «proposal» | Сумма (left_at – entered_at) для этапа proposal / Кол-во лидов, прошедших этот этап | дни | ≤ 3 дня | Расчёт на основе регистра истории статусов |

## 3. Границы MVP (Scope / Out of scope)
**Включено (In scope):**
- Справочник «Лиды» с полями: источник, дата создания, текущий статус, ответственный.
- Импорт данных из CSV в справочники и регистр истории статусов (идемпотентно).
- Прототип канбан-доски для визуального перемещения лидов по статусам (UI-макет).
- Отчёт «Воронка продаж» с отображением количества лидов на каждом этапе и конверсии.
- Расчёт KPI (конверсия и длительность) с выводом на дашборд.
- Мок-REST API (эндпоинт `/leads`) для демонстрации интеграции.

**Исключено (Out of scope):**
- Интеграция с реальной телефонией, почтой, мессенджерами.
- Работа с персональными данными (ПДн) – только синтетика.
- Полноценная система прав доступа (роли моделируются упрощённо).
- Drag-and-drop в 1С (достаточно макета в UI-прототипе).

## 8. Acceptance-критерии (основные)
*Полный список – в файле `acceptance.md`.*
- Импорт CSV не создаёт дубликатов при повторной загрузке.
- Отчёт «Воронка» отображает количество лидов на каждом этапе.
- Канбан-доска (прототип) позволяет «перетащить» лид из `new` в `qualified`.
- Расчёт конверсии `new→won` на наборе T1 совпадает с эталоном (golden) с точностью ±1%.
