# MoSCoW - приоритизация требований v0.1

## Must have (Обязательно для MVP)
- **M1** – Справочник «Лиды» (lead_id, created_ad, source, current_stage, owner).
- **M2** – Отчёт «Воронка продаж» с визуализацией количества лидов на каждом этапе (new → qualified → proposal → won / lost) и расчётом конверсии между этапами.
- **M3** - Отчет должен позволять фильтровать данные по: источнику, менеджеру, периоду
- **M4** – Импорт данных из CSV-файлов в справочник «Лиды» и регистр истории статусов (`lead_stages`). Импорт должен быть идемпотентным (повторная загрузка не дублирует записи).
- **M5** – Прототип канбан-доски (3-5 экранов), отображающий лиды по статусам. Возможность «перетаскивания».
- **M6** – Расчёт KPI с выводом на дашборд.
- **M7** - Разделение ответственности по ролям ≥3 (менеджер, руководитель отдела продаж, аналитик). Должны быть разные права на просмотр и изменение статусов.
- **M8** - Экспорт данных в CSV.
- **M9** - Мок REST API (эндпоинт /leads) с поддержкой GET и POST, возвращающий тестовые JSON объекты

## Should have
- **S1** - Форма для сбора лидов
- **S2** - Визуальные элементы в отчете: диаграмма воронки и столбчатая диаграмма длительностей

## Could have
- **C1** - Интеграция UI с конфигурацией 1С через HTTP сервис
- **C1** - При отказе выбор причины отказа

## Won't have
- **W1** - интеграция с внешними системами (телефония, email, чат-боты).
- **W2** - Работа с реальными персональными данными.
- **W3** - Адаптивный интерфейс (без мобильной версии).
