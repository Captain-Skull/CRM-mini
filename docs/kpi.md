# Ключевые показатели эффективности (KPI)

## 1. Конверсия по всем стадиям
**Название:** Stages conversion rate
**Цель:** Оценить эффективность работы с лидами на всём пути.

**Формула:**
SCR = (Количество лидов, прошедших стадию X) / (количество лидов на предыдущей стадии) * 100%

**Единица измерения:** проценты (%).  
**Целевое значение:** ≥ 60% (на синтетических данных T1).  
**Источник данных:**  
- Таблица `leads` (дата создания, финальный статус)  
- История `lead_stages` (для проверки корректности последовательности)

**Способ проверки:** отчёт 1С выводит конверсию автоматически.

## 2. Конверсия new -> won
**Название:** Conversion Rate (new → won)  
**Цель:** Оценить эффективность работы с лидами на всём пути.

**Формула:**
CR = (Количество лидов, достигших статуса 'won') / (Количество лидов, созданных за период) * 100%

*Примечание:* учитываются лиды, созданные в выбранном периоде.

**Единица измерения:** проценты (%).  
**Целевое значение:** ≥ 20% (на синтетических данных T1).  
**Источник данных:**  
- Таблица `leads` (дата создания, финальный статус)  
- История `lead_stages` (для проверки корректности последовательности)

**Способ проверки:** отчёт 1С выводит конверсию автоматически.

## 3. Средняя длительность этапа
**Название:** Average Time  
**Цель:** Выявить узкие места в процессе; чем дольше этап, тем сложнее лиду принять решение и больше вероятность отказа.

**Формула:**
AvgTime = Σ (left_at - entered_at) для всех лидов, прошедших этап / Количество таких лидов

*left_at* – дата перехода на следующий этап или финальный статус;  
*entered_at* – дата входа в этап.

**Единица измерения:** дни (округление до 0.1).  
**Целевое значение:** ≤ 3 дня (на синтетических данных T1).  
**Источник данных:** регистр (или таблица) `lead_stages`.  
**Способ проверки:** расчёт в отчёте 1С или отдельном виджете; сверка с эталоном `/data/C03/golden/avg_proposal_duration_T1.csv`.
